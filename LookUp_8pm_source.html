30 =head1 NAME
31 
32 Bio::EnsEMBL::LookUp
33 
34 =head1 SYNOPSIS
35 
36 # default creation using latest public release of Ensembl Genomes
37 my $lookup = Bio::EnsEMBL::LookUp->new();
38 
39 =head1 DESCRIPTION
40 
41 This module is a helper that provides additional methods to aid navigating a registry of >6000 species across >25 databases.
42 It does not replace the Registry but provides some additional methods for finding species e.g. by searching for species that
43 have an alias that match a regular expression, or species which are derived from a specific ENA/INSDC accession, or species
44 that belong to a particular part of the taxonomy.
45 
46 There are a number of ways of creating a lookup but the simplest is to use the default setting of the latest publicly
47 available Ensembl Genomes databases:
48 
49  my $lookup = Bio::EnsEMBL::LookUp->new();
50 
51 Once a lookup has been created, there are various methods to retreive DBAdaptors for species of interest:
52 
53 1. To find species by name - all DBAdaptors for species with a name or alias matching the supplied string:
54 
55  $dbas = $lookup->get_by_name_exact('Escherichia coli str. K-12 substr. MG1655');
56 
57 2. To find species by name pattern - all DBAdaptors for species with a name or alias matching the supplied regexp:
58 
59  $dbas = $lookup->get_by_name_exact('Escherichia coli .*);
60 
61 3. To find species with the supplied taxonomy ID:
62 
63  $dbas = $lookup->get_all_by_taxon_id(388919);
64 
65 4. To find DBAs for all descendants of a node:
66 
67  $dbas = $lookup->get_all_by_taxon_branch(511145);
68 
69 The retrieved DBAdaptors can then be used as normal e.g.
70 
71  for my $gene (@{$dba->get_GeneAdaptor()->fetch_all_by_biotype('protein_coding')}) {
72  print $gene->external_name."\n";
73  }
74 
75 If a DBAdaptor is not likely to be used again, it should be disconnected to avoid running out of connections
76 (a disconnected DBAdaptor can be used again without having to be explicitly reconnected):
77 
78  $dba->dbc()->disconnect_if_idle();
79 
80 Once retrieved, the arguments needed for constructing a DBAdaptor directly can be dumped for later use e.g.
81 
82  my $args = $lookup->dba_to_args($dba);
83  ... store and retrieve $args for use in another script ...
84  my $resurrected_dba = Bio::EnsEMBL::DBSQL::DBAdaptor->new(@$args);
85 
86 =head2 Local implementation
87 
88 The default implementation of LookUp is a remoting implementation that uses a MySQL database backend
89 to look up genome information. The previous implementation loaded an internal hash from either a JSON file
90 (remote or local) or by processing the contents of the Registry.
91 
92 This implementation is still available, but has been renamed Bio::EnsEMBL::LookUp::LocalLookUp and should
93 be constructed directly.

136 sub new {
137  my ($class, @args) = @_;
138  my $self = bless({}, ref($class) || $class);
139  ($self->{lookup}, $self->{registry},
140  $self->{url}, $self->{file}
141  ) = rearrange([qw(lookup registry url file)], @args);
142  if (defined $self->{url} ||
143  defined $self->{registry} ||
144  defined $self->{file})
145  {
146  warning(
147 q/Direct construction of local or url\/file-based LookUp deprecated.
148  Use Bio::EnsEMBL::LookUp->new() for new remoting implementation or
149  use Bio::EnsEMBL::LookUp::LocalLookUp->new() directly for previous implementation/
150  );
151  $self->{lookup} =
152  Bio::EnsEMBL::LookUp::LocalLookUp->new(@args);
153  }
154  if (!defined $self->{lookup}) {
155  $self->{lookup} = Bio::EnsEMBL::LookUp::RemoteLookUp->new(@args);
156  }
157  return $self;
158 }

170 use vars '$AUTOLOAD';
171 sub AUTOLOAD {
172  my ( $self, @args ) = @_;
173  (my $method = $AUTOLOAD) =~ s/^.*::(\w+)$/$1/ ;
174  return $self->{lookup}->$method(@args);
175 }

178 sub uniq {
179  my @out = ();
180  my $keys = {};
181  for my $v (@_) {
182  if(!exists $keys->{$v}) {
183  push @out,$v;
184  $keys->{$v} = 1;
185  }
186  }
187  return @out;
